## üìä Sheet Details

### Normal Template (7 Sheets)

**1. Overview**
- Project Name, Description, Client
- Start/End Date, Budget, Status, Owner

**2. UI Specifications**
- Screen components with validation rules
- Priority levels, Master/Detail relationships
- Business rules for each screen

**3. API Specifications**
- HTTP methods (GET, POST, etc.)
- Endpoints, request/response payloads
- Business rules for each API

**4. LLM Prompts**
- AI model configuration
- Temperature settings (0.0-2.0)
- Input variables and expected outputs

**5. Database Schema**
- Table names, field names, data types
- Primary/Foreign keys
- Nullable fields and descriptions

**6. Technology Stack**
- Frontend, Backend, Database, Cache
- Versions and repository URLs
- Rationale for each technology

**7. Traceability Matrix**
- Links requirements to implementation
- Shows which UI/API/LLM implements each requirement
- Status tracking (Implemented, In Progress, Planned)

---

### Agentic Template (10 Sheets = 7 + 3 Agent Sheets)

**Additional 3 Sheets**:

**8. Agent Architecture**
- Agent ID, Name, Type (Autonomous/Reactive/Proactive)
- Primary Role and Capabilities
- Dependencies and Communication Protocol

**9. Agent Configuration**
- Configuration parameters for each agent
- Parameter types (string, integer, float, boolean)
- Required flag for mandatory parameters

**10. Agent Tasks**
- Task ID, Task Name, Task Type
- Input/Output data specifications
- Success criteria and error handling

---

### Multi-Agentic Template (10 Sheets)

**Same structure as Agentic but with**:
- 4 agents instead of 3
- Healthcare-specific configuration
- HIPAA compliance parameters
- Data encryption settings
- Insurance verification workflows
- Multi-agent coordination examples
